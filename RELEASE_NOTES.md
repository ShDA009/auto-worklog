## auto-worklog v0.0.3

Добавлен учет времени для созданных в течении дня задач.

> Ранняя версия — возможны ошибки и изменения в API и поведении.

## Возможности

- **plan** — построение плана рабочей нагрузки на день или диапазон дат
- **apply** — отправка worklog в Jira через Tempo Timesheets API после подтверждения
    - Поддержка диапазонов дат с автоматическим исключением выходных
    - Автопривязка встреч к issue по ключу в теме (ODP-123)
    - Дедупликация: повторный apply не создаёт дубли
    - Распределение оставшегося времени по Jira issue (weighted по пересечению интервалов)
    - Настраиваемый вид работ (Руководство, Разработка, Аналитика, Тестирование)
    - Режим менеджера: автораспределение неиспользованного времени

## Интеграции

- **EWS** — Exchange Web Services (NTLM, SOAP/XML)
- **Jira REST API** — получение задач и активности из changelog
- **Tempo Timesheets API** — создание и дедупликация worklogs

## Платформы

- macOS arm64 / amd64
- Linux amd64
- Windows amd64

## Использование

```sh
./worklog plan                         # план на сегодня
./worklog plan -d 2026-03-27           # план на дату
./worklog plan -d 2026-03-24:2026-03-28  # диапазон (без выходных)
./worklog apply -d 2026-03-27          # списание в Jira
```
